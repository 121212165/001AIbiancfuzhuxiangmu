## 执行摘要

### 产品定位升级

**v1.x**: 探索者-思考者双AGENT工具系统
**v2.0**: 五AGENT完全自动化自主知识工作者

### 核心升级价值

```
从: 用户 → 配置 → 探索 → 手动挖掘 → 手动整理 → 手动总结
到: 配置一次 → 自动探索 → 自动挖掘 → 自动整理 → 自动输出 → 持续优化
                                ↓                      ↓
                          完全无人值守             自我进化学习
```

### 关键升级点

1. **新增Organizer Agent (整理者)**: 结构化知识，构建知识图谱
2. **新增Outputter Agent (输出者)**: 生成用户可消费的报告
3. **完全自动化闭环**: 输出反馈优化探索策略
4. **知识积累机制**: 系统越用越聪明
5. **用户友好输出**: 不再是原始数据，而是可消费的知识

## 1. 产品定位 2.0

### 1.1 产品愿景

**成为你的自主知识工作者 - 7x24小时不间断为你发现、整理、传递价值**

### 1.2 从工具到自主知识工作者的升级

#### v1.x - 工具模式
```
用户主动发起
    ↓
手动配置参数
    ↓
系统执行探索
    ↓
用户手动挖掘
    ↓
用户手动整理
    ↓
用户手动总结
```

**特点**:
- 用户每个环节都需要参与
- 系统是效率工具，需要驱动
- 输出是原始数据，需要二次加工
- 每次运行都是独立的，不积累知识

#### v2.0 - 自主工作者模式
```
用户配置一次 (初始种子、关注领域、输出偏好)
    ↓
系统自动运行 (定时触发)
    ↓
Explorer → Evaluator → Thinker → Organizer → Outputter
    ↓
自动生成报告 (日报/周报/洞察简报)
    ↓
反馈优化探索策略
    ↓
[循环] 系统持续进化
```

**特点**:
- 配置一次，永久运行
- 完全无人值守
- 输出是可消费的知识
- 系统积累知识图谱，越用越聪明

### 3.2 Explorer Agent (探索者) - P0

**优先级**: P0 (必须完成 - 保留自v1.x)
**状态**: 已实现，需优化

**功能描述**: 从信息海洋中发现原始内容

**核心职责**:
1. **多源搜索**: Arxiv、Web、API、RSS
2. **智能策略**: 随机、图遍历、边缘探索
3. **去重过滤**: 避免重复内容
4. **质量预筛**: 快速跳过明显低质内容

**用户价值**:
- ✅ 自动发现用户可能错过的内容
- ✅ 突破信息茧房，发现边缘但有价值的内容
- ✅ 节省用户手动搜索的时间

**输入参数**:
```python
{
    "config": {
        "seeds": ["machine learning"],  # 初始种子
        "max_iterations": 20,           # 最大探索迭代
        "strategy": "diverse",          # 探索策略
        "sources": ["arxiv", "web"]     # 数据源
    }
}
```

**输出结果**:
```python
{
    "status": "success",
    "data": {
        "discovered": [Item],           # 新发现的内容
        "sources_used": ["arxiv"],      # 使用的数据源
        "metrics": {
            "total_found": 15,
            "new_items": 12,
            "duplicates_removed": 3,
            "time_taken": 5.2
        }
    }
}
```

**v2.0升级点**:
- 接收来自Outputter的反馈，优化探索方向
- 根据知识图谱中的gap，主动探索缺失领域
- 支持动态调整探索策略

**验收标准**:
- AC1: 支持至少2个数据源
- AC2: 可以直接import使用
- AC3: 单次探索耗时 < 30秒
- AC4: [NEW] 响应反馈优化策略，响应时间 < 1分钟

### 3.4 Thinker Agent (思考者) - P1

**优先级**: P1 (应该完成 - 保留自v1.x)
**状态**: 已实现，需优化

**功能描述**: 从低质量内容中深度挖掘隐藏价值

**核心职责**:
1. **挖掘宝石**: 单个内容的隐藏价值
2. **综合洞察**: 多个内容的组合价值
3. **发现关联**: 跨内容的隐藏联系
4. **生成种子**: 为探索者提供新方向

**用户价值**:
- ✅ 不浪费任何内容，即使是低质量内容也可能有价值
- ✅ 发现人类难以察觉的跨领域关联
- ✅ 生成有洞察力的知识，而不是简单的聚合

**工作模式**:
- `mine_gems`: 从低质量内容中挖掘被低估的宝石
- `synthesize`: 综合多篇内容生成洞察
- `discover_connections`: 发现内容间的隐藏关联
- `auto`: 自动选择最佳模式

**输出结果**:
```python
{
    "status": "success",
    "data": {
        "hidden_gems": [Insight],       # 隐藏宝石
        "synthesis": [Insight],          # 综合洞察
        "connections": [Connection],     # 发现的关联
        "new_seeds": [str],              # 新探索方向
        "metrics": {
            "gems_found": 2,
            "insights_created": 3,
            "connections_discovered": 5,
            "time_taken": 8.5
        }
    }
}
```

```python
    updated_graph = updated_graph.add_edges(discover_relations(new_items))

    # 持久化
    storage.save_knowledge_graph(updated_graph)

    return updated_graph
```

### 5.4 自我优化机制

```python
class SelfOptimizingOrchestrator:
    def __init__(self):
        self.explorer = Explorer()
        self.evaluator = Evaluator()
        self.thinker = Thinker()
        self.organizer = Organizer()
        self.outputter = Outputter()
        self.analytics = Analytics()

    def run_auto_loop(self):
        while True:
            # 1. 执行完整流程
            result = self.run_full_cycle()

            # 2. 收集指标
            metrics = self.analytics.collect_metrics(result)

            # 3. 分析性能
            performance = self.analytics.analyze_performance(metrics)

            # 4. 自动调优
            if performance["quality"] < 0.7:
                self.evaluator.adjust_threshold(stricter=True)
            if performance["diversity"] < 0.5:
                self.explorer.adjust_strategy(more_diverse=True)
            if performance["user_satisfaction"] < 0.8:
                self.outputter.adjust_format(more_concise=True)

            # 5. 等待下次运行
            time.sleep(24 * 3600)  # 每天运行
```
